## Security

- **Kafka вҶ” Fabric**: SSL/mTLS вҖ” mutual certificate authentication
- **Certificate management**: Azure Key Vault (PEM format)
- **Network**: Private VNet with delegated subnet for Fabric connector
- **No secrets in code**: Certificates generated by Bicep deployment script

## Prerequisites

- Azure CLI вүҘ 2.60
- Azure subscription with Contributor access
- Fabric capacity (F SKU or Trial) in West Europe
- SSH key pair

## Deployment

### 1. Deploy Infrastructure

```bash
# Create resource group
az group create --name rg-kafka-direct-01 --location westeurope

# Deploy (generates certs, creates VM, Key Vault, VNet)
az deployment group create \
  --resource-group rg-kafka-direct-01 \
  --template-file infra/main.bicep \
  --parameters adminSshPublicKey="$(cat ~/.ssh/id_ed25519.pub)"
```

### 2. Setup Kafka on VM

```bash
VM_NAME=$(az deployment group show -g rg-kafka-direct-01 -n main \
  --query properties.outputs.kafkaVmName.value -o tsv)

# Run setup script (downloads certs from KV, starts Kafka with mTLS)
az vm run-command invoke \
  --resource-group rg-kafka-direct-01 \
  --name "$VM_NAME" \
  --command-id RunShellScript \
  --scripts @kafka/setup-kafka.sh
```

### 3. Start Event Generator

```bash
az vm run-command invoke \
  --resource-group rg-kafka-direct-01 \
  --name "$VM_NAME" \
  --command-id RunShellScript \
  --scripts "cd /data/kafka && docker run -d --name event-generator \
    --network host \
    -e KAFKA_BOOTSTRAP_SERVERS=localhost:9092 \
    -e KAFKA_TOPIC=iot-events \
    -e EVENTS_PER_SECOND=5 \
    event-generator-b"
```

### 4. Configure Fabric

Follow the steps in [fabric/README.md](fabric/README.md) to:
1. Create Streaming vNet Data Gateway
2. Create Eventhouse + KQL Database
3. Create Eventstream with Apache Kafka source (mTLS)

## Verify

```bash
# Check Kafka is running with SSL listener
az vm run-command invoke -g rg-kafka-direct-01 -n "$VM_NAME" \
  --command-id RunShellScript \
  --scripts "docker exec kafka /opt/kafka/bin/kafka-topics.sh \
    --bootstrap-server localhost:9092 --list"

# Check mTLS listener works (using client certs)
az vm run-command invoke -g rg-kafka-direct-01 -n "$VM_NAME" \
  --command-id RunShellScript \
  --scripts "docker exec kafka /opt/kafka/bin/kafka-console-consumer.sh \
    --bootstrap-server localhost:9092 --topic iot-events --max-messages 3 --from-beginning"
```
